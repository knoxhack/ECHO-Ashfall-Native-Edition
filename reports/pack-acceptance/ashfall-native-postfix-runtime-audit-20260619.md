## Terminal Mission-Filter Hotfix

- Root cause for the earlier mission-filter symptom was narrowed to ScreenCore mission browser filtering, not missing MissionCore JSON. MissionCore loaded, but Ashfall-scoped route filters could hide MissionCore-owned rows because the rows came through the generic `echomissioncore:missions` provider.
- `TerminalScreenCoreDataProviders` now tags rows with `missionNamespace` and treats Ashfall namespace filters as valid matches for MissionCore rows whose mission id or chapter namespace is `echoashfallprotocol`.
- The stale-provider reset remains in place, but the Ashfall namespace alias is no longer treated as stale when matching MissionCore rows exist.
- Regression coverage was added in `ModGameTests.terminalScreenCoreNativeMissionFallback`: opening the Ashfall provider route keeps `echoashfallprotocol:ashfall_protocol` as the provider filter and still shows an `echoashfallprotocol:*` MissionCore row.
