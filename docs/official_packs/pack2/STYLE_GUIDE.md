# Pack #2 Style Guide

Working name: ECHO: Foundry Protocol.

## Identity

Pack #2 should feel like an operational factory restoration sim inside the ECHO ecosystem: precise, readable, industrial, and systems-forward.

It is not a ruined survival march. It is a controlled but damaged facility coming back online.

## Theme

- Base ThemeCore theme: `pack2_theme`.
- Secondary compatible theme: `tech_console`.
- UI mood: dark graphite panels, cyan instrumentation, amber caution states, green production health.
- Avoid orange-dust survival composition as the dominant identity.

## Writing Voice

- Direct facility language: commission, route, calibrate, inspect, queue, stabilize.
- Keep mission copy operational instead of heroic.
- Keep lore present but secondary to clear factory tasks.

## Interface Rules

- Terminal first screen should show power, throughput, depot requests, machine health, and next commissioning task.
- Index entries should read like operator manuals.
- HoloMap layers should separate power, logistics, machines, and access routes.
- Lens scans should explain what a machine is doing and why it is blocked.

## Content Rule

No final gameplay claims until the first 30 minutes loop is implemented and tested.
