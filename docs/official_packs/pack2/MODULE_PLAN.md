# Pack #2 Module Plan

Working name: ECHO: Foundry Protocol.

## Required Modules

- `echocore`: shared services, registries, validation contracts.
- `echonetcore`: sync and packet foundation.

## Recommended Modules

- `echoterminal`: facility dashboard, production queue, route guidance.
- `echoindex`: machine, power, logistics, and commissioning docs.
- `echomissioncore`: first factory restoration mission chain.
- `echopowergrid`: generators, storage, grid routing, and power diagnostics.
- `echoindustrialnexus`: processor, factory, and machine-route identity.
- `echomultiblockcore`: larger production cells and facility validation.
- `echologisticsnetwork`: depot requests, storage, and delivery loops.
- `echoholomap`: factory floors, loading docks, substation map layers.
- `echolens`: scan profiles for machines, conduits, depots, and hazard panels.
- `echoruntimeguard`: factory budget guidance and performance-safe scaling.
- `echothemecore`: industrial UI skin foundation.
- `echotutorialcore`: first-use hints and guided route reminders.
- `echosoundcore`: low-key factory ambience, objective stingers, and warning cues.
- `echoworldcore`: data-driven region and hazard labels for the starter route.

## Optional Modules

- `echodatacore`: persistent facility state and team progress.
- `echoplayercore`: server utility for factory crews.
- `echorecovery`: server recovery support.
- `echowiki`: player-facing reference surface.
- `echoscreencore`: richer UI surface once the pack is beyond draft.
- `echoblockworks`: structural palette support.
- `echoconvoyprotocol`: later freight route expansion.
- `echonpcore`: later operator/crew briefing support.
- `echotextureforge`: creator-side visual QA only.

## Pack Boundary

Generic improvements belong in reusable modules only when they help multiple packs. Facility lore, Pack #2 progression text, factory floor routes, and branding stay inside the Pack #2 profile/template/content surfaces.
